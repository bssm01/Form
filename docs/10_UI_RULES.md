# Instagram-Inspired UI Rules

Version: 1.0

---

# Purpose

The application should feel modern, lightweight, and effortless to use.

Take inspiration from the usability principles of modern social applications.

Do NOT copy Instagram's branding, colors, layouts, assets, or proprietary design.

Create an original interface inspired by the same design philosophy.

---

# Core Design Principles

The UI should feel

- Clean
- Minimal
- Spacious
- Fast
- Calm
- Familiar
- Premium

Every element must have a purpose.

Remove everything unnecessary.

---

# Layout

Use a centered layout.

Maximum Width

1100px

Authentication Pages

420px

Profile Page

700px

Content should never stretch across the entire screen.

Large empty margins improve readability.

---

# Navigation

Navigation should always stay simple.

Desktop

---------------------------------

Logo

Spacer

Profile

Logout

---------------------------------

Mobile

Logo

↓

Profile

↓

Logout

No sidebars.

No complicated menus.

No dropdowns unless necessary.

---

# White Space

Use generous spacing.

Spacing improves readability.

Never place components too close together.

Users should never feel overwhelmed.

---

# Card Design

Cards should appear almost flat.

Small border.

Very soft shadow.

Large border radius.

Enough internal padding.

Avoid heavy shadows.

Avoid thick borders.

---

# Authentication Pages

Everything centered.

Small logo.

Large title.

Helpful description.

Simple form.

One primary button.

One secondary link.

Nothing else.

Example

LOGO

Welcome Back

Sign in to continue

Email

Password

Login Button

Don't have an account?

Register

# Animations

Animations should almost disappear.

Allowed

Fade

Scale

Slide

Hover

Duration

150-200ms

No bounce.

No spinning.

No flashy effects.

---

# Page Transitions

Transitions should be subtle.

Quick fade.

Avoid dramatic transitions.

---

# Loading

Prefer Skeletons.

Avoid fullscreen spinners.

Buttons should show inline loading.

---

# Error Messages

Simple.

Friendly.

Example

Please try again.

Avoid technical language.

---

# Empty States

Description

Primary Button

Never display blank pages.

# Shadows

Very soft.

Almost invisible.

---

# Borders

Thin.

---

# Scroll

Smooth.

Natural.

Never trap scrolling.

---

# User Feedback

Every action provides feedback.

Click

↓

Loading

↓

Success

↓

Toast

Users should never wonder if something happened.

---

# Visual Hierarchy

Each screen must answer

Where am I?

↓

What is this page?

↓

What is the primary action?

↓

What can I do next?

Everything else is secondary.

---

# Component Consistency

Buttons always look identical.

Inputs always look identical.

Cards always look identical.

Spacing always follows the design system.

No exceptions.

---

# Simplicity Rules

If a feature can be built with

2 components

do not build 5.

If a layout can use

1 card

do not use 4.

If text can be shorter,

make it shorter.

---

# Interaction Principles

The interface should feel responsive.

Every click should have feedback.

Hover states should be subtle.

Focus states should be visible.

Disabled states should be obvious.

---

# Final Design Philosophy

The interface should disappear.

Users should focus on their task,

not on the design.

Minimalism is not about removing features.

Minimalism is about removing distractions.

Every pixel should have a purpose.
